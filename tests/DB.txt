Nota sobre preferencia_usuario: Como un usuario tiene un set de preferencias, en lugar de crear una subcolección (como decía el SQL), en NoSQL es mucho más eficiente embeberlo como un Mapa (Objeto) dentro del mismo documento del usuario para ahorrar costos de lectura:
"preferencias": {
  "tipo_procesamiento_preferido": "commemoration", // String
  "consentimiento_publicacion": true               // Boolean
}

  "tipos_aceptados": ["largeAppliance", "smallAppliance"], // Array (Mejor que un CSV como en SQL)

}

Subcolección de Dispositivos: /tracking
Como un dispositivo tendrá múltiples estados a lo largo del tiempo (relación 1:N que crece), esto sí debe ser una subcolección.
// Ruta: /dispositivos/{id_dispositivo}/tracking/{id_tracking}

  "etapa": "delivery",              // String
